# Roadmap Bug Bounty

## Fase 0 — Base técnica

- HTTP / HTTPS
- Métodos, headers, cookies, sessions
- JavaScript básico para leer lógica cliente
- HTML / DOM / requests
- Burp Suite nivel operativo
- DevTools y debugging

## Fase 1 — Vulnerabilidades probables

- XSS
- IDOR
- Broken Access Control
- CSRF
- Auth bypass
- Information disclosure
- Business logic flaws

## Fase 2 — Recon y superficie

- Subdomain enumeration
- Content discovery
- Parámetros ocultos
- Wayback / gau / katana
- JS recon
- API mapping

## Fase 3 — Hunting real

- Programas con buen scope
- Triage de targets
- Repetición de checks
- Notas por programa
- Reportes reproducibles

## Fase 4 — Primer dinero

- Meta: primer bounty válido
- Meta: acumular $500
- Meta: llegar a $1000
- Analizar qué vectores funcionaron más
- Especializarse en 1–2 bugs
